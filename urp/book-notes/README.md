# Book Notes

This folder stores detailed notes from books mined for this project.

## Purpose

When a book is read, notes from it go here in a dedicated file. The goal is:
1. Preserve the specific passages, frameworks, and techniques that matter
2. Enable traceability (any claim in the main files can be checked against its source)
3. Keep the main topic files focused rather than bloated with quoted material

## Integration Pattern

When content from a book informs a main file:
- The relevant content gets synthesized into the appropriate topic file (01-08)
- The citation points back to the book
- The detailed notes and quotes stay here for reference

Example: if *The Urban Homestead* has a technique for grey water capture, the technique gets described in `02-systems-design.md` (or `04-inventories.md` if it's a specific household good) with a citation like "[Coyne & Knutzen, *The Urban Homestead*, p. X]". The full context and adjacent ideas stay in `urban-homestead-notes.md` here.

## Note Structure

Each book notes file should include:
- **Bibliographic info** (author, title, edition, publisher, year)
- **Relevance to this project** — a short paragraph on why this book matters for the Atlanta urban renter design
- **Chapter-by-chapter or topic-by-topic notes** with page numbers and direct quotes where warranted
- **Techniques and frameworks** extracted — actionable items
- **Cross-references** to which topic files in the main project each section informs
- **Reading status** — partial / complete / dormant
